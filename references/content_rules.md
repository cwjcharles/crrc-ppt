# 内容规则 / content_rules

本文件展开 [SKILL.md](../SKILL.md) Step 1 工作流的全部内容约束、标题禁词与输出模板。

## 1. 内容保真原则

- **核心事实不遗漏、不虚构、不篡改**。表达可以压缩，但不得改变原意。
- 信息过多时，优先保留 **1 个核心结论 + 必要佐证**；次要说明可降级为注释或辅助卡。
- 信息冲突时**分别呈现**，不替用户判断真假。
- 多语言内容**保持原文语种**。
- 正文尽量使用短句或要点；超过 3 句话时必须改写为短要点，**不得把长段落强行塞入卡片**。
- 适合可视化的数据优先用表格、柱状图、折线图、饼图、时间轴或简单内联 SVG 图表；信息不足时不强行图表化。

## 2. 标题约束

### 2.1 标题写法

- 标题必须来自**原文关键词、事实对象、动作或结论**。
- 优先**事实型短标题**，不为显得高级而创造抽象口号。
- 标题控制在 **6-14 个汉字**。
- 内容只压缩信息，**不扩写立意**，不加入原文没有的因果、价值判断或战略意义。

### 2.2 标题禁词表

**严禁**把原文没有的、类似空泛概念词作为标题核心：

| 禁词 |
|---|
| 赋能 |
| 引领 |
| 跃迁 |
| 重塑 |
| 筑基 |
| 聚势 |
| 焕新 |
| 生态 |
| 闭环 |
| 底座 |
| 抓手 |
| 范式 |
| 协同升级 |

> 这些词在咨询输出里高频出现但通常空洞无信息量。如果原文确实使用了这些词且是用户原意，可以保留；否则**禁止**作为标题主词。

### 2.3 标题示例

| 不可（空泛） | 可（事实型） |
|---|---|
| 数字化赋能业务升级 | 三大业务系统已上线 |
| 渠道生态全面焕新 | 经销商网络覆盖 12 省 |
| 客户体验深度跃迁 | NPS 从 32 提升至 58 |
| 战略闭环全面构建 | Q3 完成 5 项关键举措 |

## 3. Step 1 输出模板

输出模板见 [../SKILL.md](../SKILL.md) "Step 1 输出格式"段，不在此重复。注意【待用户确认】块必须声明默认采用 **方案 A：中车红灰配色**，并提示可改用 **方案 B：彩色配色**。

## 4. Step 1 输出禁区

在 Step 1 阶段**不得**输出以下内容：

- 具体配色（不要写 "#C70019" 这样的色值）
- 图标技术细节（不要写 SVG path）
- PPTX 兼容讨论
- 边框策略
- 卡片配色策略字段（多个并列卡片时尤其要省略）——但 Step 1 末尾【待用户确认】处需声明"默认方案 A：中车红灰配色，可改方案 B：彩色配色"，这是确认闭环的一部分，不算技术细节

这些都在 Step 2 阶段按 [design_spec.md](design_spec.md) 与 [pptx_compat.md](pptx_compat.md) 执行。

## 5. 确认闭环规则

- 用户明确表示 **"确认 / 同意 / 满意 / 可以继续 / 没问题 / OK / 通过"** 等积极语义 → 进入 Step 2。
- 用户提任何修改意见、补充要求、不满意反馈 → **视为未确认**，必须先修订方案并再次输出 Step 1，等待下一轮确认。
- 用户回复模糊（如"嗯"、"看着不错"、"挺好的"但未明确"确认"）→ 礼貌追问"是否可以按此方案进入 HTML 生成？"。
- 用户若选择 **方案 B：彩色配色**，视为对配色的选择，须在 Step 1 方案中记录该选择后再确认，不影响内容确认进度。
- 循环直到用户明确确认为止。

## 6. icon 决策

- **默认不添加 icon**。
- 如果用户在 Step 1 反馈中提出"加 icon"，或卡片信息明显需要图形辅助识别（如风险/状态/趋势）：
  - 在 Step 1 中向用户**确认 icon 用法**（哪些卡片加、加哪个语义的 icon）。
  - Step 2 实现时，icon 必须为**完整静态内联 SVG**，不得用图标字体或外链 SVG。
