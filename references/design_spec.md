# 设计规范 / design_spec

本文件是 [SKILL.md](../SKILL.md) 中"风格定位"的详细展开。色彩、Bento 布局、字体、视觉元素的硬性规则都在这里。

## 1. 风格定位

- 定位为"央企工业咨询风"：**稳重、清晰、克制、可信**，避免营销海报化、潮流化或过度装饰。
- 高级感来自 Bento Grid 主次层级、留白、标题秩序和数据表达，**不依赖**复杂背景、强阴影或炫技效果。
- 每页必须有一个明确视觉锚点：主标题、主卡、核心数字或关键图表之一；不得让所有卡片视觉权重完全相同。
- 卡片装饰必须服务信息结构。红条、侧边线、徽章、底座和几何元素**不得成为无意义装饰**。

## 2. 色彩规范

### 2.1 页面与卡片底色

- **页面底色**：模板底色固定为 `#FFFFFF`（白底）。所有 `.slide` 必须设 `background: #FFFFFF`，不得替换为深色或彩色整页背景。
- **卡片底色**：所有 Bento 卡片、数据卡、指标卡和说明卡默认使用 `#FFFFFF`，靠 **边框 `#B4B5B5`** 和 **轻微阴影** 与页面区分。
  - 方案 A（默认）下不得使用彩色底作为卡片背景。
  - 当 `#B4B5B5` 用作卡片标题区或大色块背景时，文字用 `#000000` 或 `#717071`，避免白字低对比。

### 2.2 文本色

| 用途 | 颜色 | 备注 |
|---|---|---|
| 正文 | `#000000` | 主要内容 |
| 次要说明 | `#717071` | 注释、辅助、来源 |
| 重点强调 | 加粗 **或** 中车红 `#C70019` | 二选一，不叠加；同页克制 |

直接落在 `#FFFFFF` 上的标题、正文和数字**必须满足对比度要求**。

### 2.3 结构色

| 用途 | 颜色 |
|---|---|
| 默认边框 | `#B4B5B5` |
| 弱分隔线、坐标轴、网格线、辅助线 | `#717071`、`#B4B5B5` |
| 少量结构导视 | `#C70019`（克制） |
| 轻微阴影 | `box-shadow: 0 2px 12px rgba(0,0,0,0.08)`（**仅一层**） |

### 2.4 CRRC 强调色板

```
中车红 #C70019   ← 品牌主色：关键结论、核心数字、结构导视、少量品牌强调
深灰   #717071
浅灰   #B4B5B5
蓝     #00678F
橙     #DA9252
绿     #92B346
```

**中车红定位**：`#C70019` 既是品牌主色，也是图表多系列第一色。用于关键结论、核心数字、结构导视和少量品牌强调，**同页使用应克制**，不大面积铺红。

**强调用途**：强调色仅用于标题区、顶部条、左侧强调线、关键数字、图表系列或少量结构导视。在方案 A 下**不得改变整张卡片底色**。

**强调约束**：
- 同一卡片内**优先只选一种强调方式**。避免标题区、顶部条、左侧强调线和关键数字同时堆叠。
- 灰色（`#717071`、`#B4B5B5`）用于信息层级：次级说明、边框、分隔线、网格线和弱化参照。

### 2.5 对比度硬约束

- 任意正文/卡片标题色与其背景**必须有明显反差**。
- 强调色区域（标题区、顶部色条、左侧强调线所在容器）若用中车红 `#C70019` 等深色作底，**其上文字必须使用 `#FFFFFF`**。
- 当 `#B4B5B5` 作大色块底时，文字用 `#000000` 或 `#717071`（浅灰底不可配白字）。
- 自检必须目视确认：
  - 深色文字不落在深色强调区上
  - 浅色文字不落在浅色底上
  - `.slide` 底色为白底，卡片靠边框+阴影区分
  - 局部强调不降低正文可读性

## 3. 图表配色

### 3.1 多系列取色顺序

```
#C70019  →  #717071  →  #B4B5B5  →  #00678F  →  #DA9252  →  #92B346
```

该规则**仅适用于同一图表内的多个数据系列**，不自动适用于模块卡片、Bento 卡片、指标卡或页面分栏。

### 3.2 图表选型

- 占比数据分类较多时，**优先考虑条形图、表格或分组展示**（比饼图更易比较）。
- 是否使用饼图/环形图由数据复杂度、版面空间和表达目标综合决定，不作强制限制。
- 饼图、环形图、进度环**禁止用 `stroke-dasharray` / `stroke-dashoffset` 模拟**，必须用真实 `<path>` 圆弧或扇区几何。无法可靠计算时降级为柱状图、条形图、表格或数字卡。

### 3.3 卡片色板 vs 图表色板

卡片强调色板与图表多系列色板可共用色值，但**用途独立，互不替代**。图表多系列按 §3.1 取色；模块卡片仅在 Step 1 确认"方案 B：彩色配色"后才可按彩色方案上色（见 §7）。

## 4. Bento 布局

### 4.1 间距与边距

| 元素 | 数值 | 说明 |
|---|---|---|
| 画布安全边距 | **24-40 px** | 页面四周留白；信息密集时取下限 |
| 卡片间距 | **16-24 px** | 卡片之间的缝隙；信息密集时取下限 |
| 卡片内边距 | **24-32 px**（硬下限 24） | **不得为提高密度而缩小**，否则文本贴边 |
| 子容器内边距 | ≥ 16 px | 子卡内嵌套的子容器 |

**提升信息密度的正确做法**：缩小画布边距 + 缩小卡片间距 + 加大卡片本身面积 → 给卡内文字更多空间，但**保留卡片内边距与字号下限**。

### 4.2 布局原则

- **不要纵向堆叠 3 个以上卡片**。信息密集时优先横向网格、分栏或主辅卡组合。
- 卡片必须使用：
  - 白底 `#FFFFFF` 背景
  - 完整 `border: 1px solid #B4B5B5`
  - `border-radius: 24px`
  - 可加轻微阴影（见 §2.3）

## 5. 字体与文字

- 字体栈固定为 `'Microsoft YaHei', 'Inter', Arial, sans-serif`。
- 字号统一使用 **px**，不用 rem/em/vw/vh/% 之类单位。

### 5.1 字号硬约束（不是参考值）

下表是**硬约束区间**，不是"推荐值"。Step 2 自检与 `validateSlides()` 会逐元素检查。低于下限即不合规。

| 用途 | 字号区间（硬约束） | PPT 实际尺寸 | 用途说明 |
|---|---|---|---|
| 主标题 | **56-72 px** | 42-54 pt | 每页一个，居顶部 |
| 卡片标题 | **30-42 px** | 22.5-31.5 pt | 每张卡的标题行；含子卡矩阵中的子卡标题 |
| 正文 | **24-30 px** | 18-22.5 pt | 卡片内要点、说明文字、表格单元格、子卡正文 |
| 注释 | **18-22 px** | 13.5-16.5 pt | 来源标注、补充说明、短描述 |
| 关键数字 | **48-72 px** | 36-54 pt | 数据卡的核心数字 |
| 行高 | 1.25-1.5 | — | — |

> 1px = 0.75pt（1920×1080 px = 10×5.625 in PPT 画布映射）

**例外**：Export 按钮等非幻灯片 chrome 不受约束（按钮 16 px 是常态）。

### 5.2 信息装不下时怎么办（关键规则）

**严禁通过降字号挤进更多信息**。正文 24 px、注释 18 px 是硬下限。

当信息密度超过当前布局承载力时，按以下优先级处理：

1. **拆页**：把 1 页拆成 2 页，每页主题更聚焦
2. **减卡**：合并次要卡片到主卡注释区，或删除非核心卡片
3. **合并要点**：把 5 条要点合并/压缩为 3 条更高密度的短句
4. **降级**：把"独立卡片"降为"主卡内一条要点"或并入相邻卡片的注释行
5. **拓宽容器**：把卡片或矩阵的宽/高扩大，给文本更多空间。可通过 **加大卡片面积 + 缩小卡片间距**（卡片间距 16-24 px）提升信息密度。注意：**不得为提高密度而缩小卡片内边距**（卡片内边距固定 24-32 px），否则文本贴边。

以上手段用尽前不得突破字号下限。

### 5.3 其他规则

- 固定高度文本框必须**预留至少 1 行安全余量**。
- 右上角项目说明、短摘要、小数据卡、约束条件卡、表格单元格等**高风险区域**不得让文本贴近容器下边界。
- 数据卡、公式框和表格单元格中的长文本必须优先**压缩、拆行或扩大容器**，**不能降字号、不能依赖 `overflow: hidden` 裁切**来制造整齐观感。
- 抗 flex/grid 纵向撑破依赖每个子项的 `min-height: 0`；`validateSlides()` 会强制校验，详见 [pptx_compat.md §1](pptx_compat.md)。

## 6. 视觉元素

- 顶部红条、侧边强调线、标题区强调、关键数字和分隔高亮必须用**独立、浅层、可定位**的 HTML 元素表达（独立绝对定位 `div`，避免单边边框或虚线边框）。颜色优先使用中车红 `#C70019` 或 CRRC 强调色板。
- 几何装饰、徽章、标签、分隔线、图形底座，在 PPT 中应作为 shape 对象，尽量表达为独立、浅层、可定位的 HTML 元素。
- **Icon 仅在用户确认后添加**。可参考 FontAwesome、Material Icons、Lucide 的极简线性造型，但**最终必须转为完整静态内联 SVG 或稳定 HTML 文本图标**。
- 严格禁用：Emoji、外链图标字体（FontAwesome / Material 等 CDN）、外链 CSS（含 Google Fonts）、远程 SVG、运行时脚本、动态加载。

## 7. 卡片配色策略（A/B 双方案）

Step 1 如存在多个并列模块卡片，必须由用户确认卡片配色策略（默认 A）：

- **方案 A：中车红灰配色（默认）**
  - 白底、灰边框，以 `#C70019`、`#717071`、`#B4B5B5` 做**小面积强调**。
  - 主卡靠**字号、间距、强调色顶部条/左侧线**与辅助卡区分，而非靠"换个底色"。
  - 数据卡的关键数字可使用中车红 `#C70019`；标题区如使用强调色块底，文字按 §2.5 取白色或深色。

- **方案 B：彩色配色**
  - 标题区、顶部条、左侧强调线或关键数字可按 §3.1 顺序使用 `#C70019`、`#717071`、`#B4B5B5`、`#00678F`、`#DA9252`、`#92B346`，用于**模块区分**。
  - 彩色配色**只用于卡片模块区分**，不等同于图表多系列配色；**未在 Step 1 确认前不得默认使用**。

**通用约束**：
- 同页面内不同卡片**总强调色种类 ≤ 3 种**，避免色彩噪声。
- 当 `#B4B5B5` 用作卡片标题区或大色块背景时，文字用 `#000000` 或 `#717071`。
